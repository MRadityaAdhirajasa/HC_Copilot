# Backend Architecture Planning: Candidate Scoring Tracker

Dokumen ini mendefinisikan struktur database, alur data (data flow), integrasi API LLM, serta logika bisnis yang akan dijalankan di balik layar aplikasi.

## 1. Database Schema & ERD (Supabase / PostgreSQL)

Karena aplikasi ini bersifat demo dan fungsionalitasnya linier, kita akan menggunakan satu tabel utama bernama `candidates` dengan relasi flat terstruktur. Jika ke depannya dikembangkan, tabel ini bisa dipecah menjadi relasi antara tabel `candidates` dan `evaluations`.

### Tabel: `candidates`
| Nama Kolom | Tipe Data | Atribut | Deskripsi |
| :--- | :--- | :--- | :--- |
| `id` | SERIAL / INT8 | PRIMARY KEY | ID unik otomatis untuk setiap kandidat |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | Waktu pencatatan data ke database |
| `full_name` | VARCHAR(255) | NOT NULL | Nama lengkap pelamar (dari Google Form) |
| `email` | VARCHAR(255) | UNIQUE, NOT NULL | Alamat email pelamar |
| `github_url` | VARCHAR(255) | NULL | Tautan portofolio/GitHub pelamar |
| `skills_summary` | TEXT | NOT NULL | Pengalaman & keahlian teknis tulisan pelamar |
| `technical_score`| INT4 | DEFAULT 0 | Skor teknis dari Gemini API (1-100) |
| `culture_score`  | INT4 | DEFAULT 0 | Skor kecocokan budaya dari Gemini API (1-100) |
| `summary_reason` | TEXT | NULL | Penjelasan/justifikasi keputusan dari AI |
| `status`         | VARCHAR(50) | DEFAULT 'Pending' | Status: `Pending`, `Strong Hire`, `Hire`, `No Go` |

### SQL DDL Script (Untuk dijalankan di SQL Editor Supabase):
```sql
CREATE TABLE candidates (
    id GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    full_name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL,
    github_url VARCHAR(255),
    skills_summary TEXT NOT NULL,
    technical_score INT DEFAULT 0,
    culture_score INT DEFAULT 0,
    summary_reason TEXT,
    status VARCHAR(50) DEFAULT 'Pending'
);

```

## 2. LLM Core Reference (Hardcoded Criteria)

Untuk memberikan konteks pada Gemini API dalam melakukan *scoring*, backend akan menyediakan variabel konstan sebagai dasar komparasi:

* **JOB_DESCRIPTION:** Fokus pada Junior/Mid Backend Software Engineer dengan tech stack Python/Node.js, pemahaman SQL/PostgreSQL, RESTful API, dan Git workflow.
* **OFFICE_CULTURE:** Fokus pada kolaborasi tinggi, proaktif memecahkan masalah, kemauan belajar (growth mindset), dan adaptasi cepat di lingkungan fast-paced.

## 3. Gemini API Prompt Engineering Specification

Fungsi penilai (*Scoring Engine*) harus mengirimkan prompt terstruktur ke model `gemini-1.5-flash` (atau versi free tier terbaru) dan mewajibkan output berbentuk **JSON murni**.

### Struktur Prompt JSON System Instruction:

```text
You are an expert Human Capital Screener specializing in Tech Recruitment.
Your job is to evaluate a candidate based on the provided Job Description (JD) and Office Culture.

Compare this Candidate Data:
Name: {full_name}
Skills & Summary: {skills_summary}

Against these criteria:
Job Description: {JOB_DESCRIPTION}
Office Culture: {OFFICE_CULTURE}

Provide an objective assessment in JSON format with exactly these keys:
{
  "technical_score": <integer 1-100 based on skill fit>,
  "culture_score": <integer 1-100 based on value fit>,
  "summary_reason": "<2-3 sentences explaining why you gave those scores and why they got that status>",
  "status": "<Strictly choose only one: 'Strong Hire' (scores > 80), 'Hire' (scores 60-80), 'No Go' (scores < 60)>"
}
Make sure response contains ONLY valid JSON. Do not include markdown code block syntax (like ```json).

```

## 4. Main Backend Functions (Python Modules)

GitHub Copilot harus membuat fungsi-fungsi Python berikut di dalam backend:

1. **`init_supabase_client()`**
* Menginisialisasi koneksi menggunakan `supabase-py` memanfaatkan kredensial dari `streamlit.secrets`.


2. **`bulk_insert_candidates(df_pandas)`**
* Menerima DataFrame hasil unggahan CSV.
* Melakukan pembersihan data ringan (menghapus duplikasi email jika ada).
* Melakukan operasi *upsert/insert* data baru ke tabel `candidates` di Supabase dengan status awal `'Pending'`.


3. **`analyze_candidate_with_llm(candidate_id)`**
* Mengambil satu data kandidat dari database berdasarkan ID.
* Mengirimkan datanya ke Gemini API menggunakan prompt terstruktur di atas.
* Mengekstrak response JSON hasil dari Gemini.


4. **`update_candidate_evaluation(candidate_id, evaluation_json)`**
* Menyimpan kembali nilai `technical_score`, `culture_score`, `summary_reason`, and `status` dari JSON AI ke database Supabase untuk kandidat tersebut.
